CONVENZIONI DI SVILUPPO

    • Verranno seguite tutte le convenzioni della progettazione in java (camel case, metodi setter e getter conformi, classi con lettera maiuscola, etc...)

    • Ogni blocco di istruzioni all'interno di un costrutto verrà indentato con un 'tab'

    • Ogni blocco di istruzioni che è legato logicamente con altre istruzioni verrà scritto senza lasciare righe vuote (banalmente ogni blocco che può essere spiegato con un unico commento)

ORGANIZZAZIONE RISORSE

    • Ogni file di testo verrà organizzato e riposto nella cartella 'log'

    • Ogni eventuale risorsa multimediale verrà riposta nella cartella 'assets'

    • I codici sorgente verranno riposti in un'architettura di package coerente con il diagramma dei package
